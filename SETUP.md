## Part 1: 初期セットアップ（開発者・1回のみ）

### 1-1. スプレッドシートにタブを追加

既存の申込み管理スプレッドシート（`13il-Fw4...`）に **5つのタブ** を追加します。

#### タブ① `News`（お知らせ）

1行目（ヘッダー）にこの順で入力：

| A: 日付 | B: タグ | C: タイトル | D: 本文 | E: 公開 |
|---|---|---|---|---|

- **タグ列**: データの入力規則 → リスト → `INFO,NEWS,EVENT,IMPORTANT`
- **公開列**: データの入力規則 → リスト → `公開,非公開`

#### タブ② `ZoomSessions`（Zoom説明会）

| A: 日付 | B: 時間 | C: ホスト | D: URL | E: 公開 |
|---|---|---|---|---|

- **公開列**: 同上 `公開,非公開`

#### タブ③ `Exhibitors`（出展企業）

| A: 社名 | B: カテゴリ | C: 並び順 | D: 公開 |
|---|---|---|---|

- **カテゴリ列**: データの入力規則 → リスト → `IT,Food,Tourism,Real Estate,Health,Finance,Retail,Education,Other`
- **公開列**: 同上

#### タブ④ `SiteSettings`（サイト設定）

key-value 2列のシンプル構造：

| A: 項目 | B: 値 |
|---|---|
| 飲食店リンク先URL | `https://bniokinawa.com/` |
| 飲食店説明文 | 沖縄BNIメンバーが営む飲食店一覧はBNI沖縄公式サイトから |

#### タブ⑤ `Chapters`（リージョン・チャプターマスタ）

既存のリージョン/チャプター一覧（沖縄・台湾・韓国）を **このタブ名で** 配置します。
1行目ヘッダー：

| A: リージョン | B: チャプター名 | C: 備考 |
|---|---|---|

すでに別タブ名で同じデータがある場合は、タブ名を `Chapters` にリネームするか、`apps-script.gs` 内の定数 `TAB_CHAPTERS = 'Chapters'` を実際のタブ名に変更してください。

---

### 1-2. Apps Script を設置・デプロイ

1. スプレッドシートを開いた状態で **拡張機能 → Apps Script** をクリック
2. デフォルトの `function myFunction() {}` を全削除
3. `site/apps-script.gs` の中身を全文コピーして貼り付け
4. **保存**（Ctrl/Cmd + S）
5. 右上 **デプロイ → 新しいデプロイ**
6. 種類の選択 → **ウェブアプリ**
7. 設定：
   - 説明：`BFO2026 v1`（任意）
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
8. **デプロイ** をクリック → 初回はGoogleの権限承認画面が出る → 承認
9. 表示される **「ウェブアプリの URL」** をコピー
   （形式：`https://script.google.com/macros/s/AKfy.../exec`）

> **重要**: 今後 `apps-script.gs` を編集したら、毎回 **デプロイ → デプロイを管理 → 編集（鉛筆アイコン）→ バージョン「新しいバージョン」→ デプロイ** で更新してください（URLは変わりません）。

---

### 1-3. サイトの3ファイルに URL を貼る

コピーした Web App URL を、以下 **3か所** の `PASTE_HERE_AFTER_DEPLOY` を置き換えます：

1. `site/src/config.jsx`
   ```js
   WEB_APP_URL: 'https://script.google.com/macros/s/AKfy.../exec',
   ```

2. `site/register.html`（`<script>` 内の `CONFIG.WEB_APP_URL`）

3. `site/checkin.html`（`<script>` 内の `CONFIG.WEB_APP_URL`）

> **PIN コード**: `site/src/config.jsx` の `CHECKIN_PIN` と `site/checkin.html` の `CONFIG.PIN` は同じ値にしてください。デフォルトは `7707`。当日運営スタッフに伝える4桁の数字です。変更したい場合は両方更新。

---

### 1-4. Netlify にアップロード

1. https://app.netlify.com/ にログイン
2. **Add new site → Deploy manually**
3. `site/` フォルダをそのまま **ドラッグ＆ドロップ**
   - `apps-script.gs` と `SETUP.md` も含まれていますが、Netlifyに上がっても害はありません（公開されません、上げたければ除外してzipしてOK）
4. デプロイ完了後、Netlify管理画面で **Site settings → Domain management** からカスタムドメイン設定（例：`bfo2026.bniokinawa.com`）

#### 更新時

`site/` 内のファイルを書き換えたら、**Deploys → Drag and drop your site folder here** に再度ドラッグするだけ。

## Part 2: 日常運用（BNI担当者向け）

スプレッドシートを編集するだけで、サイトに自動反映されます（最大5分のキャッシュ遅延あり）。
**英語・繁体中文・韓国語への翻訳は自動** で行われます。日本語だけ入力してください。

### 2-1. お知らせを追加する

`News` タブで新しい行を追加：

| 日付 | タグ | タイトル | 本文 | 公開 |
|---|---|---|---|---|
| 2026/06/25 | NEWS | 新しい出展企業が追加されました | （任意の本文） | 公開 |

- **日付**：YYYY/MM/DD 形式
- **タグ**：プルダウンから選択（INFO/NEWS/EVENT/IMPORTANT）
- **タイトル**：日本語で入力
- **公開**：プルダウンで「公開」を選択（「非公開」だとサイトに出ません）

並び順は **日付の新しい順** で自動ソートされます。

### 2-2. Zoom説明会を追加する

`ZoomSessions` タブに行を追加：

| 日付 | 時間 | ホスト | URL | 公開 |
|---|---|---|---|---|
| 2026/06/05 | 19:00–20:00 | 座間味 ED 主催 | https://zoom.us/j/12345 | 公開 |

URLが空でもOK（その場合カードはクリックできなくなる）。

### 2-3. 出展企業を追加する

`Exhibitors` タブに行を追加：

| 社名 | カテゴリ | 並び順 | 公開 |
|---|---|---|---|
| 株式会社サーチマニア | IT | 1 | 公開 |

- **並び順**：数字。小さい順に表示。空欄なら最後尾
- **カテゴリ**：プルダウンから選択
- 1〜100社まで追加可能（10列のグリッドに自動レイアウト）

### 2-4. 飲食店の説明・リンクを変える

`SiteSettings` タブの「項目」列で：
- `飲食店リンク先URL` の値を書き換え（`https://bniokinawa.com/...`）
- `飲食店説明文` の値を書き換え（日本語で）

### キャッシュについて

サーバーは **5分間** コンテンツをキャッシュします。即時反映したい場合：

1. Apps Scriptエディタを開く
2. 関数選択 → `clearCache` → ▶ 実行
3. 数秒待ってサイトをリロード

## Part 3: 当日運用（受付スタッフ向け）

### 3-1. チェックインページの開き方

チェックインページは **トップページから「隠しコマンド」** で開きます：

1. トップページの **一番下のフッター** までスクロール
2. **「© 2026 BNI Okinawa REGION」のコピーライト文字を5回連続クリック**（3秒以内）
3. **PIN入力画面**が出る → 4桁の数字（運営から伝えられた値、デフォルト `7707`）を入力
4. 一度入力すると、その端末では次回以降パスワード不要（localStorage保存）

> **直接アクセス**：URLバーに `/checkin.html` を打つこともできますが、PIN認証は同じです。

### 3-2. 受付チェックインの流れ

1. 上部タブで **「受付チェックイン」** が選択されていることを確認
2. お客様のスマホに表示されたQRコードをカメラに **かざす**
3. 自動で読み取り → 「✓ 国吉弘孝 さん チェックイン完了」と表示・音
4. 既にチェック済みの方は **黄色の「⚠ 既にチェックイン済み」** が出ます

#### QRが読めない場合

下の「手動入力」欄に participant_id（例：`BFO2026-001`）を入力 → 送信

### 3-3. 懇親会チェックインの流れ

1. 上部タブで **「懇親会チェックイン」** に切替
2. 同じくQRをスキャン
3. 統計値が「懇親会の登録数 / 受付済み / 未受付」に切り替わります

> **重要**：懇親会の支払い状況は別途 `Participants` タブの `party_payment_status` 列で管理してください。チェックインは「来場確認」のみです。

### ログアウト

画面下部の「ログアウト」リンクで PIN を再要求するモードに戻せます（端末を他人に渡す場合）。

<a id="troubleshoot"></a>
## トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| サイトに何も表示されない（LOADING…のまま） | ブラウザのコンソールを確認。Babel パースエラーの可能性 |
| News/Zoom/出展企業が表示されない | (1) `config.jsx` の `WEB_APP_URL` が未設定 (2) Apps Script のデプロイが「全員」アクセス可能になっていない (3) 「公開」列が「公開」になっていない |
| 翻訳が反映されない | `LanguageApp.translate` の Daily Quota（無料枠）超過の可能性。1日100回程度の翻訳までは安全。`clearCache()` でキャッシュ更新できないか試す |
| 申込み送信エラー | (1) `register.html` の `WEB_APP_URL` 未設定 (2) `Participants` タブの列名が変わっている (3) Apps Script の権限承認が切れた |
| QRスキャナーが起動しない | `https://` でアクセスしないとカメラ権限が取れません。Netlifyは標準でhttpsなのでドメイン直アクセスを |
| カメラ権限が出ない | iOS Safari は初回だけブラウザ設定からカメラ許可が必要 |
| チェックインしたのにスプレッドシートに反映されない | Apps Script のデプロイ更新を忘れていないか確認（コード変更後は必ず再デプロイ） |
| キャッシュをすぐ消したい | Apps Scriptエディタで `clearCache` 関数を実行 |
| 出展企業を100社以上追加したい | `variantA2.jsx` の `gridTemplateColumns: 'repeat(10, 1fr)'` を 12〜15 列に増やす |

```
├── index.html              ← トップページ
├── register.html           ← 申込みフォーム
├── checkin.html            ← 当日受付（管理者用）
├── apps-script.gs          ← スプレッドシートにコピーするスクリプト

├── src/
│   ├── config.jsx          ← Web App URL と PIN を設定
│   ├── translations.jsx    ← 4言語の固定文言
│   ├── shared.jsx          ← 共有ロジック・SVG
│   └── variantA2.jsx       ← トップページ本体

    ├── bni-logo.png

```
